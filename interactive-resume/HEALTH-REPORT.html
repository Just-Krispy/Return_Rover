Bug Fixed

Compensation regex misses "paid" FIXED

File: lib/archer-chat.ts line 395

Problem: Regex /(salary|compensation|pay|rate)/ doesn't match "paid" — the question "How much does he get paid?" routed to QA experience instead of the compensation fallback.

Fix: Changed to /(salary|compensation|pay|paid|rate|wage|income)/ to also catch "paid", "wage", and "income" variants.

Content Gaps & Issues

Missing OG image GAP

The layout.tsx has opengraph-image.tsx which auto-generates an OG image, but there's no explicit og:image meta tag with a stable URL. Social link previews (LinkedIn, Slack, Discord) may show a blank or default image instead of a branded card. Consider adding a static public/og-image.png and referencing it in metadata, or verify the auto-generated route works at /opengraph-image.

Missing canonical URL GAP

No <link rel="canonical"> in the page. While metadataBase is set in layout.tsx, an explicit canonical helps SEO by consolidating signals (e.g., if someone links with query params). Add alternates: { canonical: 'https://interactive-resume-tan.vercel.app' } to the metadata export.

LinkedIn returns 999 from server-side INFO

LinkedIn aggressively blocks automated requests (HTTP 999). This is expected and not a real issue for recruiters clicking the link in a browser. The E2E suite skips external link validation for LinkedIn/GitHub for this reason. GitHub returns 000 from this network (likely firewall), but works in browser. Both links are valid — this is a testing-only artifact.

Upgrade Recommendations

Dark/light mode toggle HIGH

Add a theme toggle. Many recruiters browse during the day and may prefer a light theme. Tailwind v3+ makes this trivial with dark: variants. Even just respecting prefers-color-scheme via CSS would improve the first impression for light-mode users.

Project links to live demos / repos HIGH

Each project card has hook + detail text but no link. Adding a "View on GitHub" or "Live demo" link per project would let recruiters immediately verify the work exists. WiFi Privacy Scanner and PawsitiveID both have public repos — link them.

Scroll-triggered animations MED

The page is content-dense. Adding subtle fade-in or slide-up animations on scroll (via IntersectionObserver or a lib like Framer Motion) would make the experience feel more premium and help recruiters scan section-by-section instead of being hit with a wall of text.

Archer "How to use" tooltip MED

The Archer FAB says "Talk to Archer" but doesn't explain what Archer is until the dialog opens. A small tooltip or subtitle on the FAB ("AI briefing about Ryan") would increase click-through from recruiters who don't recognize the name.

Print page should match PDF content 1:1 MED

The /print route and the Ryan_Bradford_Resume.pdf are separate artifacts. If they drift, recruiters get conflicting info. Consider generating the PDF from the print route (via Puppeteer or a CI step) so they're always in sync. Add a test that compares print page sections against resume-data.ts.

Skill proficiency indicators LOW

Skill chips are flat lists. Adding subtle proficiency tiers (e.g., "core" vs "familiar" via color intensity or a small dot) would help recruiters quickly see primary vs secondary skills without cluttering the layout.

Schema.org structured data LOW

Add Person and WebPage JSON-LD structured data. This helps search engines show rich results (job title, location, employer) and is a quick SEO win. Tailor it to resume-data.ts fields.

How to Extend

Add a new project: Add it to resume-data.ts projects array. The content validation tests will automatically verify it has name, hook, detail, and tags. Add an Archer knowledge entry for it.

Add a new section: Add it to page.tsx with an id attribute. Add a nav link. Add a Playwright test for the section count and content.

Add a new Archer topic: Add a knowledge entry in archer-chat.ts. Add a routing test to both content-validation.js and scripts/test-archer-chat.js.

Run before every deploy: npm run test:all catches content drift, broken nav, missing sections, and Archer routing regressions.
